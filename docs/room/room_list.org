We need to create a queuing mechanism to properly load in the parentage for rooms.
If we do this correctly then we can build out the graph ourselves for determining
what a Space structure is. Currently we're using the assumption that the first
parent is the direct expected parent, but at some point we may want to try
to support more dynamic behavior.

Now we do this because when the Rust SDK starts, the SpaceService is still waiting
for the RoomListService to fully populate before it tries to make a DAG of the
Spaces. Even when it makes the DAG we can't really directly interact with it, so
instead we use the fact it exposes the parentage through a method named
joinedParentsOfChild. This is our key to building this 'tree' out.

Because we need the tree to calculate things from Space room list display
to calculating the notification total in the spaces, we do need to ensure
this works... and works well. So we build out our list and then pass it to our
queue here which periodically retries to populate the parentage for the given room.
After a number of fails we just simply determine that the room is an orphan and
remove it from the queue as well.

#+begin_src clojurescript :tangle ../../src/room/room_list.cljs
(re-frame/reg-event-db
 :rooms/parent-resolved
 (fn [db _]
   (update-in db [:ui :parent-refresh-counter] (fnil inc 0))))

(defonce !parent-queue (atom {}))

(defn enqueue-parent-check! [room-id]
  (swap! !parent-queue update room-id #(if % % 0)))

(defn process-parent-queue! []
  (let [queue         @!parent-queue
        db            @re-frame.db/app-db
        space-service (:space-service db)
        cache         (get db :rooms-unfiltered-cache {})]
    (when (and space-service (seq queue))
      (-> (p/all
           (map (fn [[id attempts]]
                  (if-let [room (get cache id)]
                    (-> (.joinedParentsOfChild space-service id)
                        (p/then (fn [parents]
                                  (if (pos? (alength parents))
                                    {:id id :status :success :parents parents :room room}
                                    {:id id :status :retry :attempts (inc attempts)})))
                        (p/catch (fn [e]
                                   (js/console.error "Queue FFI fail for" id e)
                                   {:id id :status :retry :attempts (inc attempts)})))
                    (p/resolved {:id id :status :drop})))
                queue))

          (p/then (fn [results]
                    (doseq [res results]
                      (when (= (:status res) :success)
                        (let [room (:room res)
                              parents (:parents res)]
                          (aset room "parents" parents)
                          (aset room "first-parent-id" (.-roomId (aget parents 0))))))

                    (let [to-keep (reduce (fn [acc res]
                                            (if (and (= (:status res) :retry)
                                                     (< (:attempts res) 3))
                                              (assoc acc (:id res) (:attempts res))
                                              acc))
                                          {}
                                          results)]
                      (reset! !parent-queue to-keep)

                      (when (some #(= (:status %) :success) results)
                        (re-frame/dispatch [:rooms/parent-resolved]))

                      (when (seq to-keep)
                        (js/setTimeout process-parent-queue! 2000)))))))))

Prior we had a very verbose and repeated diffs check, but
as our timeline, spaces, and room list all share the same
general patterns to handling diffs we can consolidate them and
call them like so.
So all we need to do is ask reframe for the current global rooms
and call the diff check against those rooms and the updates
to set the "new" global rooms. This will make our global list
refresh automatically.
Later we can apply filters on it to induce specific behavior
like creating a DM grouping.

To add onto this, but we use a mutex to ensure that the rust-sdk
doesn't ever end up in a state where the diffs aren't readily
available to refresh our window. We don't slow the rust-sdk down,
but instead establish a queue for our code to process them
and notify reframe properly.

(defn distinct-by [f coll]
  (let [!seen (atom #{})]
    (filterv (fn [x]
               (let [k (f x)]
                 (if (contains? @!seen k)
                   false
                   (do (swap! !seen conj k)
                       true))))
             coll)))


(defn safe-get [obj k-str k-kw]
  (if (map? obj)
    (get obj k-kw)
    (let [v (aget obj k-str)]
      (if (undefined? v)
        (aget obj (name k-kw))
        v))))

(defn flatten-tree [rooms-map children-map space-orders closed-drawers parent-id depth]
  (let [sdk-children    (get children-map parent-id [])
        manual-children (->> (vals rooms-map)
                             (filter #(= (safe-get % "first-parent-id" :first-parent-id) parent-id))
                             (map (fn [r] {:id        (or (safe-get r "id" :id) (safe-get r "roomId" :roomId))
                                           :is-space? (safe-get r "isSpace" :isSpace)})))

        orders-for-parent (get space-orders parent-id {})

        child-refs      (->> (concat sdk-children manual-children)
                             (distinct-by :id)
                             (sort-by #(get orders-for-parent (:id %) "zzzz"))
                             (vec))]
    (reduce
     (fn [acc {:keys [id is-space?]}]
       (let [summary       (get rooms-map id)
             item          {:id                 id
                            :is-space?          is-space?
                            :depth              depth
                            :activeRoomCallParticipants (when summary (safe-get summary "activeRoomCallParticipants" :activeRoomCallParticipants))

                            :name               (if summary (or (safe-get summary "name" :name) "Loading...") "Loading...")
                            :avatar             (when summary (safe-get summary "avatar" :avatar))
                            :unreadNotificationsCount (if summary (or (safe-get summary "unreadNotificationsCount" :unreadNotificationsCount) 0) 0)
                            :unreadMentionsCount    (if summary (or (safe-get summary "unreadMentionsCount" :unreadMentionsCount) 0) 0)}
             acc-with-item (conj acc item)]

         (if (and is-space? (not (contains? closed-drawers id)))
           (into acc-with-item (flatten-tree rooms-map children-map space-orders closed-drawers id (inc depth)))
           acc-with-item)))
     []
     child-refs)))
