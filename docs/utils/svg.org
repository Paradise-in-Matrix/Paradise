**** Preface
Scalable Vector Graphics solve an issue of having perfectly scaling 'icons' for example using vector math
to determine how it should look at different sizes.

Now rather than do them all by hand... We'll give a go at using a conventional library
as a reference.
Now at some point we may wish to fully import a library, but for now I think we
should just take the vector graphic itself and plant it here for reuse.
To add to that, the library icons could also be modified without too much effort
using Inkscape.

If we decide to change and use a dependency then they can just be shifted here.
We might also like to start actually compartmentalizing the CSS to be kept together with the
org file using it. The scope creep has really started to set in there.

**** Icon Base

So we should establish a nice icon base that we intend to use throughout.
Since we have intent to provide animated SVGs for our icons... we'll
go ahead and plan for that by providing a class we can hook to for
the CSS.
#+begin_src clojurescript :tangle ../../src/utils/svg.cljs
(defn- icon-base [{:keys [animate viewBox size] :as props} & children]
  (let [final-view-box (or viewBox "0 0 24 24")
        final-size (or size "16px")]
    (into
     [:svg (merge {:viewBox final-view-box
                   :width final-size
                   :height final-size
                   :fill "none"
                   :stroke "currentColor"
                   :stroke-width "2"
                   :stroke-linecap "round"
                   :stroke-linejoin "round"
                   :class (when animate (str "animate-" (name animate)))}
                  (dissoc props :animate :viewBox :size))]
     children)))
#+end_src
